## Features built

- World gen: terrain, caves, ores, trees, 5 biomes, BFS flood-fill lighting
- 8 world presets: Mixed, Grasslands, Forest, Desert, Tundra, Ocean, Cave World, Superflat
- Player: move, sprint, jump, gravity, collision detection
- Mining (left click, crack animation) + block placement (right click)
- Inventory: 36 slots, drag-and-drop, 2×2 and 3×3 crafting
- Creative mode: instant break, fly (double-tap Space), unlimited blocks
- Block palette: 20 block types, click → fills selected hotbar slot with 64
- Health + hunger bars
- Entities: Zombie, Creeper, Pig with night-spawn AI
- Day/night cycle + dynamic lighting overlay
- Auto-save to localStorage every 60 seconds (save format v2)
- Environment selector: pick preset → world fully regenerates with loading screen

## Hard rules (learned the hard way — do not change these)

1. **Always use `bundle.js`** — never `type="module"` imports. ES modules broke on Vercel CDN.
2. **Keyboard listeners on `window` with capture phase** — `window.addEventListener('keydown', fn, true)`. Document-level listeners get intercepted.
3. **Draw player AFTER lighting overlay** — if drawn before, player is invisible underground.
4. **Save format must have `version: 2`** — load code clears saves without it. Prevents stale-state spawn bugs.
5. **In Playwright tests, use `keyboard.down()` + wait for movement keys** — `keyboard.press()` is <1ms, game loop misses it.
